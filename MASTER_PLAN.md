# Abundance Effect: Master Plan (v1.0)

## 0. ASSETS & GLOSSARY (Справочник)
Тут зафиксированы определения, чтобы мы говорили на одном языке.
*   **Стек**: Next.js 14 (App Router), Tailwind CSS, Framer Motion, Supabase (Auth, DB, Storage), Vercel, AI (Gemini/Grok).
*   **Платформа**: PWA (Web-приложение, работающее как нативное). Mobile First.
*   **Core (Ядро)**: Неснижаемый, несгораемый капитал. Генерирует 0.0633% в день (пожизненно). При пополнении токены сжигаются, увеличивая базу.
*   **Liquid $**: Внутренний стейблкоин (1:1 к USDT). Доступен для вывода, покупок, переводов.
*   **Levels (Уровни)**: Зависят от размера Ядра. Прогрессия: $2, $4, $8, $16, $32, $64, $128... $250, $500, $1000 (10 lvl) и т.д.
*   **AI Squad**: Набор ИИ-личностей (Врач, Наставник, Друг).
*   **Avatar**: "Ai Sims" пользователя. Показывает последствия действий в будущем. Экономика аватара = Реальность × 100.
*   **AI-Core Economy (ИИ-экономика Ядра)**: Экономика, где AI координирует задания, капитал и людей, а распределение дохода и приоритетов идет пропорционально подтвержденному вкладу пользователя в систему.
*   **Reward Split (Модель наград)**: Для системных заданий обязательная часть награды всегда идет в Core; бонусная часть зависит от результата и может включать Liquid $, предметы, репутацию и доступ к более дорогим заданиям.
*   **System Starter Challenges**: Набор обязательных челленджей для новичков (низкий уровень), через которые пользователь изучает механику платформы до доступа к пользовательским задачам.
*   **Mutuality (Взаимки)**: Механика роста через взаимные сделки пользователей (покупка услуг друг у друга) с ростом чеков по мере уровня и защитой от накруток.
*   **Design System (UI/UX)**:
    *   **Style**: iOS / Telegram / Instagram (Clean, Light, Mobile-First).
    *   **Colors**: Background `#F2F2F7`, Surface `#FFFFFF`, Text `#000000` / `#8E8E93`, Accent `#007AFF`.
    *   **Components**: Rounded Cards (16px), Pill Buttons, Soft Shadows.
    *   **Development Guidelines**:
        *   **i18n First**: Все компоненты создаем с учетом языкового контекста (`useLanguage`).
        *   **Minimalism**: Минимизируем текст, где возможно. Например, в нижней панели используем только иконки без подписей (или скрываем подписи на мобильных).
        *   **Local Cache Strategy**: Используем `utils/storage.ts` для сохранения настроек на устройстве.
            *   Ключи: `app-language` (язык), `app-theme` (тема), `app-user-settings` (настройки).
            *   При старте приложения восстанавливаем состояние из кэша.
    *   **Technical Standards (Database)**:
        *   **Write Operations**: Все изменения данных (INSERT, UPDATE, DELETE) выполняем **строго через Server Actions** (`app/actions/*`). Это решает проблему разрыва соединений при сворачивании приложения.
        *   **Read Operations**: Критические данные читаем через Server Actions. Для UI-оптимизаций допускается чтение на клиенте, но с осторожностью.

## 0.55. AI-CORE CHALLENGE LOOP (Операционная модель)
Этот раздел фиксирует целевую механику ИИ-фриланс биржи, чтобы разработка шла итеративно и согласованно.

### 1. Принципы
*   Челленджи = обучение через реальные действия + реальный доход.
*   Системные челленджи на старте обязательны и формируют поведение пользователя.
*   Низкий уровень = ограниченный пул безопасных заданий; рост уровня = доступ к более дорогим и сложным заданиям.
*   Пользователь проходит путь: `системные задания -> задания системы высокого уровня -> задания от пользователей -> создание своих заданий`.

### 2. Модель наград
*   **Системные задания**:
    *   Обязательная часть: всегда начисляется в Core.
    *   Бонусная часть: зависит от качества/скорости/вклада и может начисляться в Liquid $, предметах, репутации и доступах.
*   **Пользовательские задания**:
    *   Базовая выплата идет из бюджета заказчика (escrow).
    *   Поддерживается комбинация выплат: Liquid $ + Core + предметы.
    *   Для устойчивого роста экосистемы часть награды начисляется в Core исполнителя (% комиссия платформы).

### 3. Приоритетные сферы развития пользователей
*   Привлечение новых пользователей и команд.
*   Создание кода/сервисов платформы.
*   Создание контента и growth-кейсов.
*   Привлечение средств в Core (личный и командный рост).
*   Сервисные услуги внутри экосистемы (дизайн, аудит, поддержка, обучение).

### 4. Роль AI-Core в распределении ценности
*   AI координирует спрос/предложение задач, подбирает исполнителей и цену.
*   AI считает подтвержденный вклад пользователя (деньги, задачи, качество, полезность для системы).
*   Исполнение потребностей/желаний пользователей приоритизируется пропорционально вкладу.

### 5. Механика взаимности (Взаимки)
*   Пользователи покупают услуги друг у друга на возрастающие суммы по мере роста уровней.
*   Цель: ускорить оборот внутри системы и быстрее показывать рост дохода.
*   Обязательная защита: анти-фрод/анти-накрутка (проверка реальности услуг, лимиты круговых сделок, риск-скоринг пар).

#### 1.1 ИИ-Координатор как Операционная Система (AI Operating System)
**Цель:**
Перевести ИИ из режима "чат-ответов" в режим постоянного управления циклом исполнения желаний пользователей.

**Лучшие практики (Best Practices):**
- `Memory First`: ИИ хранит не только диалоги, а операционную память системы:
  - `user_memory` (цели, навыки, паттерны действий, прогресс).
  - `system_memory` (спрос/предложение, дефициты, простаивающие ресурсы).
  - `process_memory` (что сработало/не сработало в прошлых механиках).
- `Goal Engine`: у ИИ есть постоянные измеримые цели (доход пользователя, активность, качество матчей, удовлетворенность, снижение простаивания ресурсов).
- `Continuous Loop`: ИИ работает циклом `Observe -> Analyze -> Plan -> Act -> Learn`.
- `Multi-Agent Roles`: разделение ролей между агентами (Coordinator, Economy, Growth, Social, Builder), чтобы избежать "одного универсального бота".
- `Safe Agency`: все действия ИИ проходят через policy-слой, лимиты риска и whitelist server actions.
- `Human-in-the-Loop`: для high-impact действий требуется подтверждение; low-risk действия могут выполняться автоматически.
- `Metrics-driven`: каждое действие ИИ связано с KPI и пост-анализом эффекта.

**Операционные артефакты:**
- [ ] Таблицы состояния и памяти (`events`, `user_memory`, `system_goals`, `ai_plans`, `ai_actions`, `ai_outcomes`).
- [ ] Планировщик фоновых циклов (5 минут / 1 час / 24 часа).
- [ ] Панель "AI Control Center" (цели, планы, действия, результаты, ручной override).

- 🛠️ **Заказы от пользователей**: Люди могут размещать свои челленджи/задания с оплатой в Liquid $. Например: "Сделай дизайн логотипа", "Проверь мой код", "Организуй доставку".
- 🔐 **Смарт-контракты и P2P**: Система резервирует средства. Проверку выполнения осуществляет либо сам заказчик, либо сторонние "пиры" (рецензенты) за небольшой %.
- 🤖 **AI-Копирайтер**: ИИ помогает пользователю грамотно составить объявление или описание услуги по шаблону.
- 🎓 **Аттестация навыков**: Система выдает цифровые сертификаты на основе выполненных заданий. Это повышает **Рейтинг и Репутацию**, открывая доступ к более дорогим заказам.
- 📶 **Уровневый допуск**: Пользователь может принять только задания уровня `<=` своего уровня; задания выше уровня видны, но заблокированы для принятия.
- 🔁 **Взаимки как ускоритель**: Пользователи взаимно покупают услуги внутри платформы, увеличивая подтвержденный оборот и доходы по мере роста уровней.

**2. Инвестиции нового типа: AI Core Management**
- 📈 **ИИ как управляющий активами**: Главный продукт для инвестиций — AI Core. Пользователь вкладывает ресурсы, а ИИ управляет ими для преумножения общего капитала системы.
- 💰 **Гарантированный доход**: Система обеспечивает 0.0633% в день (26% годовых). Как именно ИИ достигает этого — сложное алгоритмическое управление, о котором инвестору не нужно заботиться.
- 🤝 **Интересы пользователей**: В отличие от корпораций, ИИ Abundance работает на максимизацию благосостояния участников (совладельцев), а не на увеличение неравенства.
- ⚖️ **Пропорциональность вклада**: Приоритизация потребностей и распределение выгод в системе учитывают подтвержденный личный и командный вклад в AI-Core экономику.

**3. Самогасящиеся финансовые инструменты**
- 💳 **Интеллектуальная рассрочка**: Покупка товаров в кредит, который автоматически гасится за счет ежедневной доходности от Ядра пользователя.
- 🔄 **Арбитраж потребностей**: Пользователь может оплачивать своей внешней кредиткой покупки, нужные другим людям, а те возвращают ему средства с небольшим процентом, обеспеченным их предварительно пополненным Ядром.

**4. ИИ-ультраинструментарий (The AI Toolbox)**
- 🧰 **Все нейросети в одном месте**: ИИ Наставник постепенно получает доступ ко всем топовым инструментам (генерация изображений, кода, анализ данных, монтаж видео).
- 🎟️ **Модель использования**: Базовые функции доступны бесплатно в рамках дневного лимита. Для серьезной профессиональной работы пользователь может приобретать дополнительные токены или подписки. Начинаем с самой востребованной функции (например, AI-помощник по созданию контента/заработка).
